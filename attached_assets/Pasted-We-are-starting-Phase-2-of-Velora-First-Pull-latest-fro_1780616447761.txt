- recurring subscriptions

Build ONLY:

1. Merchant Payment Links

Inside the Merchant Dashboard, allow a merchant to create a payment link.

Fields:
- Payment title
- Description
- Amount
- Currency: SOL and USDC demo option
- Customer email or wallet address optional
- Expiry date optional

After creation:
- save payment link in Supabase
- generate unique checkout URL
- show QR code
- allow copy link
- show payment link status: Draft / Active / Paid / Expired

2. Supabase Tables

Create or update schema for:
- payment_links
- payments
- transactions

Each payment link should store:
- id
- merchant_id
- merchant_wallet
- title
- description
- amount
- currency
- status
- checkout_url
- created_at
- expires_at

3. Checkout Page

Create a checkout page where a consumer can open a payment link.

Checkout should show:
- merchant name
- payment title
- amount
- currency
- connected wallet
- payment status
- pay button

For Phase 2, prepare the transaction flow on Devnet but keep it safe and clearly marked as test/devnet.

4. Payment Confirmation Foundation

After payment action:
- show loading state
- show success state
- create payment record in Supabase
- create transaction record placeholder
- show receipt confirmation screen
