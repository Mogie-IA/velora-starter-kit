PHASE 3 — SOLANA PAYMENT PROCESSING & SETTLEMENT

First:

* Pull latest from GitHub.
* Continue from Phase 2.
* Keep GitHub sync active.
* Continue using the Velora design system.
* Use Solana Devnet only.
* Do not expose secrets, API keys, seed phrases, private keys, or service-role keys.

PHASE 3 OBJECTIVE

Transform Velora from a payment-link platform into a functioning Solana payment processor.

A customer should be able to complete an actual Devnet payment from Phantom and have Velora automatically record and confirm it.

DO NOT BUILD

* Embedded wallets
* Google Sign-In
* Apple Sign-In
* Subscription billing
* Marketplace features
* Merchant storefronts
* Analytics dashboards
* Mainnet support

Build ONLY the payment-processing layer.

========================================
PAYMENT EXECUTION
=================

On checkout:

1. Connect Phantom Wallet
2. Validate wallet connection
3. Validate payment link
4. Generate Solana transaction
5. Send transaction on Devnet
6. Capture transaction signature
7. Store transaction signature
8. Update payment status

Supported:

* SOL payments

USDC architecture may be prepared but does not need full implementation.
