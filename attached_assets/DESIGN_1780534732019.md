## Brand & Style

The design system establishes a high-performance fintech identity that bridges the gap between traditional premium finance and high-speed blockchain technology. The aesthetic is rooted in **Modern Minimalism** with a **Tactile Glassmorphism** influence, prioritizing clarity, trust, and frictionless movement.

The interface should feel airy and expansive, utilizing heavy whitespace to reduce cognitive load in complex financial workflows. By moving away from the "dark mode" tropes of crypto, the design system positions itself as a mainstream-ready financial tool, drawing on the precision of Linear and the approachable sophistication of Apple. The emotional response should be one of complete security and effortless speed.

## Colors

The palette is anchored by a high-chroma **Primary Purple**, used intentionally for key actions and brand accents. The background architecture uses a very soft lavender-tinted white to maintain a "premium" feel compared to generic pure white.

### Core Tones
- **Primary:** The core driver of the UI. Use for primary buttons, active states, and focus indicators.
- **Dark Navy:** Reserved for primary typography and deep-contrast elements to ensure accessibility and authority.
- **Background:** A cool-toned base that provides a soft canvas for floating elements.
- **Success/Warning/Error:** Standardized semantic colors for transaction statuses and system feedback.

### Gradients
- **Primary Glow:** A linear gradient from `#6D4AFF` to `#4F46E5` (Indigo) used for hero elements and high-priority cards.
- **Background Accents:** Large, low-opacity radial blurs (15% opacity) of the primary color should be placed behind key containers to create depth.

## Typography

This design system utilizes **Inter** exclusively to achieve a systematic, utilitarian, yet modern appearance. 

### Hierarchy Rules
- **Display & Headlines:** Use tight letter-spacing and bold weights to anchor the page. 
- **Numerical Data:** For wallet balances and transaction amounts, use `body-lg` or `headline-md` with `tabular-nums` enabled to ensure vertical alignment in lists.
- **Labels:** Use `label-sm` in all-caps for metadata headers or secondary navigation to provide structural contrast without increasing visual weight.

## Layout & Spacing

The design system is built on a strict **8px grid system**. All padding, margins, and component heights must be multiples of 8 to ensure a rhythmic and balanced layout.

### Grid & Breakpoints
- **Desktop (1440px+):** 12-column fluid grid. 80px side margins.
- **Tablet (768px - 1024px):** 8-column grid. 48px side margins. 
- **Mobile (<768px):** 4-column grid. 20px side margins.

### Composition
- **Whitespace:** Prioritize vertical "breathing room." Group related items with 16px or 24px spacing, while separating distinct sections with 80px+ gaps to signify shifts in context.

## Elevation & Depth

Depth is achieved through **Tonal Layering** and **Soft Ambient Shadows**. The system avoids harsh black shadows in favor of tinted, multi-layered drop shadows that make components appear as if they are floating in a 3D space.

### Layering Strategy
1.  **Background (#F8F7FF):** The infinite canvas.
2.  **Surface Level (White):** Primary containers and cards. These use a shadow: `0 4px 20px rgba(11, 16, 38, 0.04)`.
3.  **Floating Level (Glass):** Modals, dropdowns, and navigation bars. These utilize a backdrop blur (12px) and a subtle `1px` inner border of `rgba(255, 255, 255, 0.5)` to simulate a glass edge.
4.  **Interaction Level:** Hover states on cards should slightly increase the shadow spread and lift the element by `2px` via a transform.

## Shapes

The shape language is "Optimistically Rounded." It balances the friendliness of consumer fintech with the precision of a professional SaaS.

- **Large Containers (Cards):** 24px (`rounded-xl`). This creates a distinct "pod" look for transaction groups and wallet summaries.
- **Action Elements (Buttons):** 16px. This distinguishes buttons from smaller inputs and larger cards.
- **Form Elements (Inputs):** 14px. A slightly tighter radius than buttons to provide a clear visual distinction between interactive "triggers" and "fields."
- **Badges/Chips:** Full pill-shaped (999px) for status indicators.

## Components

### Buttons
- **Primary:** Gradient fill (Purple -> Indigo), white text, 16px radius.
- **Secondary:** White fill, `1px` border of `#E7E4FF`, Navy text.
- **Size:** Standard height is 48px; Large (for CTA) is 56px.

### Cards
- **Architecture:** Solid white background, 24px radius, 1px subtle border.
- **Glass Variant:** Used for sidebars or overlays. `80%` opacity white with `20px` backdrop blur.

### Inputs
- **Style:** Height 48px. 14px radius. 
- **Focus State:** `2px` solid primary purple stroke with a soft purple outer glow (4px spread).
- **Labels:** Always placed outside the input field in `label-md` weight.

### List Items
- **Transaction Row:** 72px height. Leading icon in a 40px rounded-lg container with 10% primary color tint. Trailing text for currency amounts should use `Inter` semibold.

### Chips & Tags
- **Status Tags:** Low-saturation backgrounds with high-saturation text (e.g., Success: Background 10% opacity Green, Text 100% opacity Green). Pill-shaped only.
